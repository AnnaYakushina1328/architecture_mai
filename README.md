# Система заказа такси https://www.uber.com/
## Приложение должно содержать следующие данные:
- Пользователь
- Водитель
- Поездка

## Реализовать API:
- Создание нового пользователя
- Поиск пользователя по логину
- Поиск пользователя по маске имя и фамилии
- Регистрация водителя
- Создание заказа поездки
- Получение активных заказов
- Принятие заказа водителем
- Получение истории поездок пользователя
- Завершение поездки

## 2.1. РОЛИ ПОЛЬЗОВАТЕЛЕЙ
1. Пассажир (Passenger/User)
- Создает аккаунт в системе
- Создает заказы на поездки
- Просматривает историю своих поездок
- Получает информацию об активных заказах
2. Водитель (Driver)
- Регистрируется в системе как водитель
- Получает список доступных заказов
- Принимает заказы на выполнение
- Завершает поездки
3. Администратор системы (Administrator)
- Поиск пользователя по логину
- Поиск пользователя по маске имя и фамилии
- Управление пользователями и водителями
- Мониторинг системы

## 2.2. ВНЕШНИЕ СИСТЕМЫ
Для полноценной работы системы заказа такси необходимо взаимолдействие со следующими внешними системами:

1. Платёжная система (Payment Gateway)
- Обработка платежей за поездки
- Привязка банковских карт
- Возврат средств

2. Сервис геолокации и карт (Maps/Geolocation)
- Определенеие местоположения пассажиров и водителей
- Построение маршрутов
- Расчёт расстояния и времени поездки

3. SMS-шлюз (SMS Gateway)
- Отправка кодов подтверждения при регистрации
- Уведомления о статусе заказа

4. Email-сервис (Email Service)
- Отправка подтверждений регистрации
- Чеки и квитанции о поездках
- Уведомления и рассылки

5. Сервис push-уведомлений (Push Notification Service)
- Мгновенные уведомления о новых заказах для водителей
- Уведомления пассажирам о прибытии водителя

## ## Архитектура системы
Архитектура системы описана в стиле **C4 Model** с использованием **Structurizr DSL**.

Описание архитектуры находится в файле `3.dsl`.

## Диаграммы

### C1 — Системный контекст
Контекстная диаграмма системы показывает:

- **Роли пользователей:**
  - Пассажир — создаёт заказы, отслеживает поездки, просматривает историю
  - Водитель — получает заказы, принимает и выполняет поездки
  - Администратор — управляет пользователями и мониторит систему

- **Взаимодействие пользователей с системой:**
  - Пассажир и водитель используют мобильные приложения
  - Администратор использует веб-панель управления

- **Интеграция системы с внешними сервисами:**
  - Платёжный шлюз — обработка платежей
  - Сервис карт и геолокации — маршруты, геопозиционирование
  - SMS-шлюз, Email-сервис, Push-сервис — уведомления
